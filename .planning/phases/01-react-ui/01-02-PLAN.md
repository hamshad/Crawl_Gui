       - error: string | null (error message)

       Layout (centered, max-w-4xl, p-6):
       - Header: "Crawl4ai" title with Phosphor Spider/Globe icon, subtitle "Web to Markdown"
       - URL Input Card (Card component):
         - Input field with Phosphor Link icon on left, placeholder "https://example.com"
         - Crawl button (Button, variant default) with Phosphor Spider icon
         - Button shows "Crawling..." text + spinner icon when loading, disabled state during crawl
         - Input allows Enter key to trigger crawl

       Status area (below input):
       - When loading: show Progress bar at ~30% + indeterminate pulse, Badge with "Crawling..." and Spinner icon
       - When success: Badge with "Complete" and CheckCircle icon, green variant
       - When error: Badge with "Failed" and Warning icon, destructive variant, error text below

       Markdown output card (when markdown is not null):
       - Card with "Result" header
       - Scrollable content area (max-h-96 overflow-auto) with prose styling
       - ReactMarkdown component with remarkGfm plugin
       - Apply Tailwind prose classes: prose prose-invert prose-sm max-w-none
       - Code blocks styled with bg-secondary rounded p-4 font-mono

       Log messages area (collapsible, below markdown):
       - Card with "Logs" header
       - Map over logs array, show each as muted text with colored dot per level (info=blue, success=green, error=red)

       API integration:
       - async function handleCrawl():
         - validate url not empty
         - set isLoading true, clear previous results
         - fetch('http://127.0.0.1:5001/crawl', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({url}) })
         - parse JSON response
         - if response.success: set markdown and logs
         - if !response.success: set error from response.error
         - set isLoading false
         - catch network errors → set error message

       Real-time updates approach:
       - Since Flask backend /crawl is synchronous (blocks until complete), show continuous loading state with animated progress bar
       - After crawl completes, display full markdown result
       - Future Phase 02 will add SSE streaming for true real-time updates
